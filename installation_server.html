Server installation

Until Collectd-pw has no package available, you need to compile it as explain bellow.
You can also use standard Collectd, but some amazing features as Top process will not work.

Prerequisites

To build Collectd-pw, you need some tools :

  • On RedHat like : 
    yum groupinstall 'Development tools'
yum install zlib-devel rrdtool-devel rrdtool
  • On Debian like : 
    aptitude install build-essential
aptitude install zlib1g-dev librrd-dev rrdtool libdbi0-dev
  • On Suse like : 
    zypper in --type pattern Basis-Devel
zypper in zlib-devel rrdtool-devel rrdtool zlib libart_lgpl-devel

Compile json-c

You can get json-c (version >= 0.10) from https://github.com/json-c/json-c/wiki. 

wget https://s3.amazonaws.com/json-c_releases/releases/json-c-0.10.tar.gz
tar xzf json-c-0.10.tar.gz
cd json-c-0.10
./configure --prefix=/usr
make -j
make install
cp json_object_iterator.h /usr/include/json # Bug in libjson install script, seems to be fixed in 0.11
cp /usr/lib/libjson.* /usr/lib64/ # Needed on RHEL and Suse x86_64

Compile libmicrohttpd

You can get libmicrohttpd (version >= 0.9.22 - older not tested) from http://www.gnu.org/software/libmicrohttpd/ 

wget http://ftp.gnu.org/gnu/libmicrohttpd/libmicrohttpd-0.9.22.tar.gz
tar xzf libmicrohttpd-0.9.22.tar.gz
cd libmicrohttpd-0.9.22
./configure --prefix=/usr
make -j
make install
cp /usr/lib/libmicrohttpd.so* /usr/lib64/ # Needed on RHEL and Suse x86_64
Note : on RHEL and Suse, if your system is x86_64, you need to put your libs in /usr/lib64. Add --libs=/usr/lib64 to your configure line : 
./configure --prefix=/usr --libdir=/usr/lib64

Getting sources

Two possibility :

  • Clone GitHub repo : 
    git clone -b 'release/5.4.0' https://github.com/perfwatcher/collectd-pw.git
  • Download last stable version : 
    wget http://perfwatcher.free.fr/download/collectd/collectd-5.4.0.20140625.tar.gz

Note : take a branch more recent than release/5.1

Compile and install Collectd-pw

cd collectd
./configure --enable-top --enable-cpu --enable-rrdtool \
--enable-jsonrpc --enable-notify_file --enable-basic_aggregator \
--enable-write_top \
--prefix=/usr
make -j
make install
cp src/types-perfwatcher.db /etc
  • On RedHat like : 
    cp contrib/redhat/init.d-collectd /etc/init.d/collectd
chmod a+x /etc/init.d/collectd
  • On Debian like : 
    curl -o /etc/init.d/collectd 'http://git.tokkee.org/?p=pkg-collectd.git;a=blob_plain;f=debian/collectd-core.collectd.init.d;hb=HEAD'
chmod a+x /etc/init.d/collectd
mkdir /etc/collectd/
a2enmod php5
a2enmod proxy
a2enmod proxy_http
service apache2 restart
  • On Suse like : 
    cp contrib/sles10.1/init.d-collectd /etc/init.d/collectd
chmod a+x /etc/init.d/collectd

Configure Collectd-pw

Below a sample of configuration for a server with perfwatcher >= 20130325 :
/etc/collectd/collectd.conf on Debian like
/etc/collectd.conf on RedHat and Suse like 

TypesDB     "/usr/share/collectd/types.db"
TypesDB     "/etc/types-perfwatcher.db"
Interval     60

LoadPlugin cpu
LoadPlugin df
LoadPlugin disk
LoadPlugin interface
LoadPlugin load
LoadPlugin memory
LoadPlugin processes
LoadPlugin swap
LoadPlugin top
LoadPlugin uptime

LoadPlugin network
<Plugin network>
	Listen "0.0.0.0" "25826"
	MaxPacketSize 65535
</Plugin>

LoadPlugin rrdtool
<Plugin rrdtool>
	DataDir "/var/lib/collectd"
</Plugin>

LoadPlugin jsonrpc
<Plugin jsonrpc>
	Port "8080"
	MaxClients 64
# Note : write_top and jsonrpc work together.
# This is why we need to define this path twice.
	DataDir "/var/lib/collectd"
# If you prefer to use rrdcached, uncomment and update the next line
#	RRDCachedDaemonAddress "/var/run/rrdcached/rrdcached.sock"
	RRDToolPath "/usr/bin/rrdtool"
	TopPsDataDir "/var/lib/collectd/top"
</Plugin>

LoadPlugin notify_file
<Plugin notify_file>
	DataDir "/var/lib/collectd/_notification"
# This is an include list. If you prefere an exclude list,
# you should exclude "top".
	Plugin "sysconfig"
	InvertPluginList false
	LinkLast "sysconfig"
</Plugin>

LoadPlugin write_top
<Plugin write_top>
	DataDir "/var/lib/collectd/top"
	FlushWhenOlderThanMin 30
	FlushWhenBiggerThanK 500
</Plugin>

LoadPlugin basic_aggregator
<Plugin basic_aggregator>
	Aggregators_config_file "/<path to perfwatcher>/etc/aggregator.conf"
</Plugin>
